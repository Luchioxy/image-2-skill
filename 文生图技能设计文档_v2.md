# WorkBuddy 文生图技能设计文档 v2.0

## 1. 项目概述

### 1.1 目标
创建一个集成到 WorkBuddy 的文生图技能，使用 OpenAI DALL·E 模型，为用户提供简单易用的图片生成能力。

### 1.2 核心特性
- **智能交互**：不确定时向用户提问澄清需求
- **API 密钥管理**：运行时检查，缺失时引导用户补充
- **多模型支持**：支持 DALL·E 2 和 DALL·E 3
- **技能集成**：检查现有技能，支持集成到其他技能中
- **本地化存储**：自动下载图片到本地工作目录

## 2. 技术架构

### 2.1 技能结构
```
~/.workbuddy/skills/dalle-image-generation/
├── SKILL.md                    # 技能说明文档
├── scripts/
│   ├── dalle_generator.py      # 核心生成脚本
│   ├── config_manager.py       # 配置管理模块
│   └── image_downloader.py     # 图片下载模块
└── examples/
    └── usage_examples.md       # 使用示例
```

### 2.2 核心模块

#### 2.2.1 配置管理模块 (`config_manager.py`)
- **功能**：管理 OpenAI API 密钥和用户偏好设置
- **存储位置**：`~/.workbuddy/dalle_config.json`
- **密钥检查**：运行时检查环境变量 `OPENAI_API_KEY` 或配置文件
- **安全处理**：密钥加密存储，运行时解密使用

#### 2.2.2 核心生成脚本 (`dalle_generator.py`)
- **API 调用**：封装 OpenAI DALL·E API 调用
- **模型选择**：支持 DALL·E 2 和 DALL·E 3
- **参数处理**：尺寸、质量、风格等参数
- **智能交互**：不确定时向用户提问澄清需求

#### 2.2.3 图片下载模块 (`image_downloader.py`)
- **自动下载**：生成后自动下载到本地
- **文件命名**：基于时间戳和描述的智能命名
- **格式支持**：PNG、JPEG、WebP 等格式

## 3. 功能设计

### 3.1 基础功能

#### 3.1.1 单张图片生成
```bash
# 基础用法
python dalle_generator.py "一只可爱的橘猫在阳光下打盹"

# 指定模型
python dalle_generator.py "未来城市夜景" --model dall-e-3

# 指定尺寸
python dalle_generator.py "产品展示图" --size 1024x1024
```

#### 3.1.2 批量生成
```bash
# 批量生成（从文件读取提示词）
python dalle_generator.py --batch prompts.txt

# 批量生成（命令行多提示词）
python dalle_generator.py --batch "提示词1" "提示词2" "提示词3"
```

### 3.2 高级功能

#### 3.2.1 自定义参数
- **尺寸**：256x256, 512x512, 1024x1024, 1792x1024, 1024x1792
- **质量**：standard, hd (仅 DALL·E 3)
- **风格**：vivid, natural (仅 DALL·E 3)
- **数量**：1-10 张 (DALL·E 2) 或 1 张 (DALL·E 3)

#### 3.2.2 智能交互系统
**当用户描述不明确时，技能会主动提问澄清：**
1. **场景澄清**："您希望生成什么类型的图片？产品展示、营销海报还是社交媒体内容？"
2. **风格确认**："您对风格有什么要求？写实、卡通、扁平化还是其他？"
3. **尺寸选择**："您需要什么尺寸？正方形(1024x1024)、横版(1792x1024)还是竖版(1024x1792)？"
4. **质量要求**："您追求速度优先还是质量优先？"

**智能决策逻辑：**
- 如果用户描述清晰完整，直接生成
- 如果描述模糊，根据上下文推断并确认
- 如果无法推断，向用户提问澄清

#### 3.2.3 技能集成检查
**自动检查用户现有技能：**
1. 扫描 `~/.workbuddy/skills/` 目录
2. 识别包含图片生成功能的技能
3. 向用户询问："检测到您已有以下图片相关技能：[技能列表]，是否希望将文生图功能集成到这些技能中？"
4. 如果用户同意，提供集成方案

### 3.3 结果管理
- **历史记录**：保存生成历史，支持重新生成
- **收藏功能**：标记喜欢的图片
- **导出功能**：支持导出为不同格式

## 4. 交互流程

### 4.1 首次使用流程
```
1. 用户调用技能："生成一张产品展示图"
2. 技能检查 API 密钥
3. 如果密钥缺失：
   a. 提示用户："检测到 OpenAI API 密钥未配置"
   b. 引导用户："请提供您的 OpenAI API 密钥"
   c. 用户输入密钥
   d. 技能验证密钥有效性
   e. 保存密钥到配置文件
4. 检查用户现有技能，询问是否集成
5. 询问用户具体需求（如果描述不明确）
6. 生成图片并展示
7. 询问是否满意，是否需要调整
```

### 4.2 日常使用流程
```
1. 用户输入描述："一只戴着墨镜的狗在冲浪"
2. 技能分析描述完整性
3. 如果描述不明确，向用户提问澄清
4. 自动选择最佳参数
5. 调用 API 生成图片
6. 下载图片到本地
7. 展示图片预览
8. 询问是否需要调整或重新生成
```

### 4.3 批量处理流程
```
1. 用户提供多个描述（文件或命令行）
2. 技能解析所有描述
3. 对于不明确的描述，逐一提问澄清
4. 生成图片
5. 显示进度："正在生成第 3/10 张..."
6. 所有图片生成完成后，提供汇总报告
```

## 5. 错误处理

### 5.1 常见错误类型
- **API 密钥错误**：密钥无效、过期、余额不足
- **网络错误**：连接超时、网络不稳定
- **内容审核**：提示词包含违规内容
- **参数错误**：无效的尺寸、模型等参数
- **配额限制**：API 调用频率限制

### 5.2 错误处理策略
- **自动重试**：网络错误自动重试 3 次
- **降级处理**：DALL·E 3 失败时降级到 DALL·E 2
- **用户引导**：提供清晰的错误信息和解决建议
- **日志记录**：记录错误详情，便于排查

## 6. 安全设计

### 6.1 API 密钥安全
- **加密存储**：使用 AES-256 加密存储密钥
- **运行时解密**：仅在内存中解密使用
- **访问控制**：仅当前用户可访问配置文件
- **密钥轮换**：支持定期更换密钥

### 6.2 内容安全
- **内容过滤**：自动过滤违规内容
- **审核机制**：集成 OpenAI 内容审核 API
- **用户警告**：违规内容生成时警告用户

## 7. 性能优化

### 7.1 并发处理
- **异步生成**：支持多张图片并发生成
- **队列管理**：批量生成时使用任务队列
- **进度反馈**：实时显示生成进度

### 7.2 缓存机制
- **结果缓存**：相同提示词使用缓存结果
- **智能去重**：避免重复生成相似图片
- **本地存储**：优化本地文件存储结构
